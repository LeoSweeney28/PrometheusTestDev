-- Core Concepts:
--
-- 1. Polymorphic Expressions
--    - Generate multiple semantically equivalent code forms
--    - Each run produces different but equivalent code
--    - Defeats pattern recognition and code comparison tools
--    - Zero performance impact
--
-- 2. Expression Complexity
--    - Wraps simple expressions in complex computations
--    - Makes code appear more sophisticated
--    - Configurable complexity levels (1-3)
--    - 10-30% code bloat depending on level
--
-- 3. Control Flow Inversion
--    - Inverts if conditions with swapped branches
--    - Transforms loops to unconventional structures
--    - Mixed inversion type for variety
--    - 10-20% code size increase
--
-- 4. Anti-Analysis Patterns
--    - Fake data flow (appears to modify state but doesn't)
--    - Masked conditionals (always true but looks complex)
--    - Decoy variables (unused but look important)
--    - Fake environment checks
--    - Defeats static analysis tools
--
-- Integration Points:
--
-- ExpressionComplexity Step:
--   - Modifies: All numeric, string, boolean expressions
--   - Configurable: Complexity level (1-3)
--   - Location: Before Vmify in Medium preset
--
-- ControlFlowInversion Step:
--   - Modifies: If statements, while/for loops, conditions
--   - Configurable: Inversion type (mixed, early_return, negative_condition)
--   - Location: Beginning of Strong preset
--
-- AntiAnalysis Step:
--   - Injects: Fake patterns, decoy variables, checks
--   - Configurable: Injection points (global, function, block)
--   - Location: Before final steps in Strong preset
--
-- Performance Characteristics:
--
-- Code Bloat:
--   - ExpressionComplexity Level 1: 10-15%
--   - ExpressionComplexity Level 2: 20-25%
--   - ExpressionComplexity Level 3: 30-35%
--   - ControlFlowInversion: 10-20%
--   - AntiAnalysis: 5-15%
--   - Total (Strong): 40-60%
--
-- Execution Overhead:
--   - ExpressionComplexity: +5-10%
--   - ControlFlowInversion: +10-15%
--   - AntiAnalysis: +2-5%
--   - Total: +20-40% (acceptable for most applications)
--
-- Analysis Tool Effectiveness:
--
-- Against IDA/Ghidra: ⭐⭐⭐⭐ (Excellent)

--   - Makes data flow tracking impossible
--   - Fake patterns cause confusion

-- Against Luadec/Unluac: ⭐⭐⭐⭐⭐ (Outstanding)

--
-- Against Debuggers: ⭐⭐⭐ (Good)

--
-- Against ML/Binary Ninja: ⭐⭐⭐⭐ (Very Good)
--   - Defeats learned patterns
--   - Expression complexity confuses classifiers
--   - Anti-analysis creates data poisoning

--
-- Medium Preset (Moderate Obfuscation):
--   - Adds ExpressionComplexity (Level 1)
--   - Code bloat: +10-15%
--   - Recommended for: General purpose protection
--
-- Strong Preset (High Obfuscation):
--   - Adds ControlFlowInversion (mixed)
--   - Adds ExpressionComplexity (Level 2)
--   - Adds AntiAnalysis (aggressive)
--   - Code bloat: +40-60%
--   - Recommended for: High-value assets requiring maximum protection
--
-- Advanced Features:
--
-- Polymorphic Module:
--   - generateValueForms(value) - Multiple forms of same value
--   - generateConditionalForms(cond, true, false) - Polymorphic conditions
--   - generateLoopForms(var, max, body) - Polymorphic loops
--   - generateFunctionForms(name, args, body) - Polymorphic functions
--
-- Complexity Module:
--   - complexifyNumber(expr) - Arithmetic wrapping
--   - complexifyString(expr) - String concatenation wrapping
--   - complexifyVariable(expr) - Variable access wrapping
--   - complexifyCondition(expr) - Boolean condition wrapping
--   - injectJunkComputation(scope) - Meaningless code injection
--
-- AntiAnalysis Module:
--   - createFakeDataFlow() - Fake state modifications
--   - createMaskedConditional(code) - Mask real conditions
--   - createDebugCheck() - Fake debugging patterns
--   - createFakeExternalDependency() - Fake dependency checks
--   - createDecoyVariable(scope) - Unused but important-looking vars
--   - generateMisdirectingName() - Confusing variable names
